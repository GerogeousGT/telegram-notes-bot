# Telegram Notes Bot

Личный Telegram-бот для сохранения заметок и транскрибации аудио/видео.

## Что умеет

- Текстовые сообщения → сохраняет в `Распределение/сообщения/`
- Голосовые → транскрибирует через AssemblyAI → `Распределение/транскрипты/`
- Документы → `Распределение/документы/`
- Изображения → `Распределение/изображения/`
- Ссылки на видео (Instagram, YouTube, TikTok) → скачивает и транскрибирует

## Структура

```
telegram-notes-bot/
├── bot.py            — точка входа, хендлеры
├── config.py         — конфигурация (читает из .env)
├── file_saver.py     — сохранение файлов по папкам
├── transcriber.py    — транскрибация через AssemblyAI
├── sync_manager.py   — логирование и статистика
├── restart-bot.ps1   — перезапуск бота на VPS
├── sync-bot.ps1      — синхронизация данных с VPS (rclone)
├── .env.example      — шаблон переменных окружения
└── Распределение/    — данные (в git не попадает)
```

## Настройка

1. Скопируй `.env.example` → `.env` и заполни, **или** добавь ключи в корневой `c:\ClaudeCode\AI\.env`
2. Установи зависимости: `pip install -r requirements.txt`
3. Запусти: `python bot.py`

## Переменные окружения

| Переменная | Обязательная | Описание |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | да | Токен от @BotFather |
| `ADMIN_ID` | да | Telegram User ID администратора |
| `ASSEMBLYAI_KEY` | да | API-ключ AssemblyAI для транскрибации |
| `DATA_FOLDER` | нет | Путь к папке данных (по умолчанию `./Распределение`) |
| `DEBUG_MODE` | нет | Включить debug-режим (`true`/`false`) |

## Деплой на VPS

Бот задеплоен на VPS. Скрипты управления:

```powershell
# Перезапустить бота на VPS
.\restart-bot.ps1

# Забрать накопленные данные с VPS на локальный диск
.\sync-bot.ps1
```

Для `sync-bot.ps1` нужен настроенный `rclone` с remote `vps-bot`.

## Команды бота

| Команда | Действие |
|---|---|
| `/start` | Приветствие |
| `/help` | Справка |
| `/status` | Статус и путь к папке данных |
| `/sync` | Статистика за сессию |
| `/log` | Последние 10 записей лога |
| `/list` | Последние 10 сохранённых файлов |
