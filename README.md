# Telegram Notes Bot

Личный Telegram AI-ассистент: ведёт диалог, сохраняет заметки по команде, ищет по ним, транскрибирует и анализирует голосовые/видео.

## Структура проекта

```
telegram-notes-bot/
├── bot.py                  — точка входа, инициализация и запуск
├── config.py               — конфигурация (читает из .env, валидирует при старте)
├── requirements.txt
├── .env.example            — шаблон переменных окружения
├── handlers/
│   ├── commands.py         — /start /help /status /sync /log /list
│   ├── messages.py         — text, voice, audio, document, photo, video, video_note
│   └── utils.py            — проверка прав доступа
├── services/
│   ├── file_saver.py       — сохранение файлов по папкам
│   ├── transcriber.py      — транскрибация через AssemblyAI
│   └── sync_manager.py     — логирование и статистика
├── scripts/
│   ├── restart-bot.ps1     — перезапуск бота на VPS
│   └── sync-bot.ps1        — синхронизация данных с VPS (rclone)
└── Распределение/          — данные (в git не попадает)
    ├── сообщения/
    ├── транскрипты/
    ├── документы/
    ├── изображения/
    └── ссылки/
```

## Что умеет

### AI-режим (требует DEEPSEEK_API_KEY)

| Действие | Пример |
|---|---|
| Диалог | Любой текст — AI отвечает |
| Сохранить заметку | "Сохрани это в заметки" |
| Сохранить в категорию | "Сохрани в категорию рецепты: [текст]" |
| Поиск по заметкам | "Найди про встречу с Иваном" |
| Создать категорию | "Создай категорию идеи для бизнеса" |
| Список категорий | "Какие у меня категории?" |
| Анализ транскрипта | После расшифровки голосового — автоматически |

### Медиа (всегда)

| Тип | Действие |
|---|---|
| Текст | AI-диалог / сохранение (если нет ключа) |
| Голосовое | Транскрибирует → сохраняет → AI анализирует |
| Аудио | Транскрибирует → сохраняет → AI анализирует |
| Видео / круглое | Транскрибирует → сохраняет → AI анализирует |
| Документ | Сохраняет в `документы/` + подпись рядом |
| Фото | Сохраняет в `изображения/` + подпись рядом |
| Ссылка на видео | Скачивает и транскрибирует (YouTube, Instagram, TikTok) |

## Настройка

1. Скопируй `.env.example` → `.env` и заполни, **или** добавь ключи в корневой `c:\ClaudeCode\AI\.env`
2. Установи зависимости: `pip install -r requirements.txt`
3. Запусти: `python bot.py`

## Переменные окружения

| Переменная | Обязательная | Описание |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | да | Токен от @BotFather |
| `ADMIN_ID` | да | Telegram User ID администратора |
| `ASSEMBLYAI_KEY` | да | API-ключ AssemblyAI |
| `DEEPSEEK_API_KEY` | нет | API-ключ DeepSeek (включает AI-режим). Бесплатный tier: [platform.deepseek.com](https://platform.deepseek.com) |
| `DATA_FOLDER` | нет | Путь к папке данных (по умолчанию `./Распределение`) |
| `DEBUG_MODE` | нет | `true`/`false` |

Если обязательная переменная не задана — бот падает при старте с понятным сообщением.

## Команды бота

| Команда | Действие |
|---|---|
| `/start` | Приветствие и список возможностей |
| `/help` | Справка |
| `/clear` | Сбросить историю диалога с AI |
| `/status` | Статус и путь к папке данных |
| `/sync` | Статистика за сессию |
| `/log` | Последние 10 записей лога |
| `/list` | Последние 10 сохранённых файлов |

## Деплой на VPS

```powershell
# Перезапустить бота на VPS
.\scripts\restart-bot.ps1

# Забрать накопленные данные с VPS на локальный диск
.\scripts\sync-bot.ps1
```

Для `sync-bot.ps1` нужен настроенный `rclone` с remote `vps-bot`.
VPS-параметры (хост, пользователь, путь) читаются из env vars или используют дефолтные значения.
